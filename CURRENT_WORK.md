# 현재 작업 인수인계

마지막 갱신: 2026-06-19

## 현재 상태

- 기본 기능 구현과 GitHub Pages 배포가 완료되어 있다.
- 배포 저장소는 이름 끝에 하이픈이 있는 `math-record-maker-`이다.
- 로컬 저장소의 원격 주소와 기본 브랜치는 정상적으로 연결되어 있다.

## 최근 완료 작업

- AI 프롬프트 강조사항에 눈에 띄는 `선택사항` 배지와 비워 두면 기본 규칙이 적용된다는 안내를 추가했다.
- 학생별 작성 항목 추가 버튼 아래에 탐구 주제, 선택 동기, 맡은 역할, 문제 해결 과정, 활용한 수학 개념, 자료 분석 방법, 발표·산출물 특징, 피드백 반영 등의 예시를 표시했다.
- 새 작성 항목 입력칸의 예시 문구가 추가 순서에 따라 다양하게 바뀌도록 했다.
- 상단 우측에 `내 소중한 세특이 날아갈까 걱정된다면?` 전체 데이터 보관 패널을 추가했다.
- 기존 수행평가 전용 백업을 `전체 저장하기·전체 불러오기`로 바꾸고 수행평가, 작성 항목, AI 강조사항, 학급별 학생 입력, 성취수준, 역량, 분량, 문체 기준을 하나의 JSON 파일에 저장하도록 확장했다.
- 전체 불러오기는 다른 수행평가를 기존 목록과 합치고 같은 수행평가의 학급 자료를 복원하며, 현재 자료가 있으면 불러오기 전에 자동 안전백업을 먼저 내려받는다.
- 백업 파일에 학생 정보가 포함될 수 있다는 안전 보관 안내를 표시했다.
- 수행평가 등록·수정 화면에 선택 입력 방식의 `AI 프롬프트 강조사항`을 추가했다.
- 강조사항은 해당 수행평가의 개별 AI 프롬프트와 학급 5명 묶음 프롬프트에 공통으로 적용되며, 학생 원본 자료에 근거가 있을 때만 반영하도록 안전 규칙을 함께 넣었다.
- 기존 수행평가와 이전 JSON 백업은 강조사항이 없는 상태로 자동 호환되며, 새 백업에는 강조사항도 함께 저장된다.
- 개별·학급 작성의 희망 분량을 간결하게 약 150자, 보통 약 250자, 상세하게 약 450자로 조정했다.
- 화면 안내, 학생별 AI 원본 자료, 학급 묶음 프롬프트, 오프라인 보조 초안의 분량 기준을 같은 값으로 통일했다.
- 수행평가 등록 화면에 `학생별 작성 항목` 구성 기능을 추가했다.
- `학생 관찰 근거(학생 활동 내용)`은 필수로 고정하고, `학생의 진로`와 `교사의 평가`는 기본 선택 항목으로 제공한다.
- 선택 항목은 삭제할 수 있고, 교사가 이름을 정한 새 작성 항목을 최대 10개까지 추가할 수 있다.
- 수행평가별로 설정한 항목만 학생 입력 화면에 나타나며 초안과 AI 프롬프트에도 반영된다.
- 기존 저장 데이터와 JSON 백업은 기본 선택 항목을 자동 적용하고, 이전 단일 추가 항목도 새 형식으로 변환한다.
- 수행평가 설명은 세특에 전부 옮기지 않고 필요한 배경만 짧게 반영하도록 생성 규칙을 조정했다.
- 학생 관찰 내용이 충분하면 핵심 행동 중심으로 요약하고, 부족하면 입력된 사실과 수행평가 맥락 안에서 분량에 맞게 보완한다.
- 짧게·보통·상세하게 설정별 문장 분량 상한을 두고 근거 없는 공통 문장을 제거했다.
- `강조할 수학 역량`을 선택 항목으로 바꾸고, 선택하지 않으면 역량 문장을 억지로 추가하지 않도록 했다.
- AI 활용의 기본 흐름을 `수행평가별 프롬프트 1회 → 학생별 원본 자료 반복 복사` 방식으로 변경했다.
- 학생별 AI 자료에는 앱이 가공한 초안을 넣지 않고 관찰 근거, 선택 작성 항목, 역량, 수준, 희망 분량만 구조화해 담는다.
- 앱의 자체 생성 결과는 `오프라인 보조 초안`으로 명확히 구분하고 보조 기능으로 유지했다.
- 수행평가 등록의 학생별 작성 항목에서 필수 항목과 선택 항목 이름의 글자 크기·굵기·색상을 통일했다.
- 학생 세특 만들기 하단에 `① 수행평가별 프롬프트 → ② 학생 원본 자료` 복사 순서를 하나의 권장 AI 흐름으로 묶었다.
- 오른쪽 오프라인 보조 초안 영역의 크기와 시각적 강조를 줄이고, 오타·문맥 문제 때문에 AI 사용을 권장한다는 안내를 항상 표시했다.
- 기존 개별 세특 화면을 보존하면서 `학급 일괄 작성` 화면을 새로 추가했다.
- 학급 화면은 수행평가별 작성 항목에 맞춘 표 편집, 엑셀 탭 구분 표 붙여넣기, 최대 40명 입력, 브라우저 자동 저장을 지원한다.
- 관찰 근거가 입력된 학생을 5명씩 묶고, 각 묶음에 수행평가·성취기준·작성 규칙을 다시 포함한 새 AI 대화방용 프롬프트를 생성한다.
- 교사용 학생 구분은 AI 자료에서 제외하고 `학생 01` 같은 익명 번호만 사용한다.
- 첫 결과의 마음에 드는 문장을 `문체 기준 잠금`에 저장하여 이후 묶음의 문장 길이·어조·구성을 맞출 수 있게 했다.
- 5명씩 새 대화방을 쓰는 이유와 문체 기준 잠금이 필요한 이유를 학급 화면에 설명했다.
- `MADE BY` 배지의 대비를 높이고 문구를 `© 인창고 수학왕 chunoring`으로 변경했다.
- 홈의 **한눈에 보는 사용방법** 영역을 더 작고 간결하게 조정했다.
- 홈의 **한눈에 보는 사용방법** 4단계 카드를 안내 전용으로 바꾸고 클릭 동작과 손가락 커서를 제거했다.
- 학급 입력을 `방법 1. 엑셀 표 한 번에 붙여넣기`와 `방법 2. 직접 입력하기`로 구분해 둘 중 하나만 선택하면 된다는 안내를 추가했다.
- 선택한 수행평가의 작성 항목에 맞는 `.xlsx` 양식을 내려받고, 작성한 엑셀 파일을 다시 불러와 최대 40명의 자료를 표에 반영할 수 있게 했다.
- 학급 표의 `강조 역량`을 직접 입력이 아닌 5개 수학 역량 다중 선택 방식으로 변경했다.
- 엑셀 업로드 결과가 학생별 수행 정보 표에 반영된다는 안내를 추가하고, 업로드 뒤 해당 표로 이동해 빠진 내용과 역량을 보완하도록 흐름을 연결했다.
- 개별·학급 AI 프롬프트가 동일한 공통 작성 규칙을 공유하도록 통합해 한쪽에서 규칙이 누락되지 않게 했다.
- 권장 서술어, 피해야 할 내면 표현과 과거·미래형 종결, 관찰 근거가 필요한 긍정어휘 기준을 두 프롬프트에 모두 포함했다.
- 상단 작업 메뉴를 `수행평가 등록 → 개별 세특 만들기 또는 학급 일괄 작성`의 선택 구조로 바꿔 두 작성 화면을 순서대로 해야 한다는 오해를 없앴다.
- 내려받은 엑셀 양식이 열리지 않던 문제를 수정했다. 워크시트 XML 요소를 Excel 표준 순서로 재배치하고 다운로드 주소를 충분히 유지하도록 보완했다.
- 학급 일괄 작성 표와 엑셀 양식의 첫 열은 자유 입력 가능한 `번호 / 구분`으로 유지하고, 처음 5개 행에는 수정 가능한 기본값 1~5를 넣었다. 이 값은 AI에 복사하지 않는다.
- 등록된 수행평가 카드의 이동 버튼을 `개별 세특 만들기`와 `학급 세특 만들기` 두 개로 나눠 선택한 평가를 각 화면에 바로 적용하도록 했다.
- 홈의 권장 AI 작성 흐름을 `개별 세특 작성`과 `학급 세특 작성` 두 트랙으로 나누고, 학급 트랙에 5명 단위 복사·묶음별 새 대화방·문체 기준 잠금 순서를 안내했다.
- `문체 기준 잠금`에 선택사항 표시를 추가하고, 입력하지 않아도 기본 문체 규칙이 적용된다는 안내를 학급 화면과 홈 흐름에 명시했다.
- 학급 일괄 작성에 관찰 근거가 있는 학생의 오프라인 보조 초안을 한 번에 생성·편집·전체 복사하는 기능을 추가하고 AI 사용 권장 및 교사 검토 안내를 표시했다.
- 엑셀 업로드와 표 붙여넣기 사이의 `또는` 표시를 확대해 둘 중 하나만 선택하는 구조를 강조했다.
- 문체 기준 잠금을 학급 화면 맨 아래로 옮기고 `나만의 문체 또는 마음에 드는 문체 붙여넣기`로 바꿨으며, 입력·수정 후 묶음 프롬프트를 반드시 다시 복사하도록 안내했다.
- 오프라인 보조 초안이 지나치게 짧고 단순해지던 조건을 보완했다. 수행평가 맥락, 관찰 과정, 입력에서 확인되는 활동 확장, 선택 역량, 진로, 교사 평가, 성취 수준을 분량에 맞게 풍부하게 조합한다.
- 개별 세특과 학급 일괄 작성의 오프라인 보조 초안이 동일한 `composeOfflineRecord` 생성 규칙을 사용하도록 확인하고, 학급의 모든 학생에게 개선된 생성 방식을 함께 적용했다.
- 오프라인 보조 초안 첫 문장에 등록된 수행평가 활동 내용을 직접 반영하고, 후보 문장이 부족할 때 수행평가와 관찰 근거에 기반한 보완 문장을 사용해 선택한 분량의 최소 범위에 가깝게 작성하도록 수정했다.
- 오프라인 보조 초안의 성취 수준이 마지막 문장에만 반영되던 문제를 수정했다. 매우 우수·우수·보통·성장 중에 따라 수행함, 분석함, 설명함, 정리함, 기록함, 적용함 등 초안 전체의 서술어와 표현 강도가 달라진다.
- 학생 관찰 근거가 초안에 그대로 복사되던 문제를 수정했다. 관찰 문장을 수행평가명과 발표·산출물 작성·자료 분석·주제 탐구·문제 해결 등의 실제 활동 과정에 연결해 재구성하며, AI 프롬프트에도 원문 단순 복사를 금지하는 규칙을 추가했다.
- 간결형과 보통형의 분량 차이가 작던 문제를 수정했다. 간결형은 수행평가 활동 설명을 짧게 요약하고 핵심 관찰 2개 중심으로 구성하며, 보통형은 활동 맥락·관찰 과정·보완 문장을 더 충분히 사용한다.
- 기존 초안이 열린 상태에서 관찰 근거·선택 작성 항목·역량·성취 수준·분량을 바꾸면 초안이 자동으로 다시 생성되도록 수정했다. 오프라인 초안 만들기 버튼도 폼 제출에 의존하지 않는 명시적인 생성 버튼으로 변경해 재생성 누락을 방지했다.
- 간결하게 등 분량 선택 카드의 투명 라디오 입력이 클릭을 받지 못할 수 있던 스타일을 수정했다. 카드 전체를 실제 입력의 클릭 영역으로 만들어 초안 작성 전후 모두 안정적으로 선택된다.
- 간결형에서 선택 역량이나 확장 문장이 없을 때 빈 문장 후보가 분량 계산기에 전달되어 개별·학급 초안 생성이 중단되던 오류를 수정했다. 모든 분량에서 빈 후보를 먼저 제거한 뒤 문장을 다듬는다.
- **AI로 더 고급스럽게** 영역을 더 크고 눈에 띄게 강조했다.
- 모바일 화면을 위한 820px, 520px 반응형 스타일을 반영했다.
- 이전 CSS 캐시를 피하도록 `index.html`의 `home.css` 주소에 버전 값을 추가했다.
- 최근 반영 커밋은 `Update home.css`, `Update index.html`이다.

## 다른 컴퓨터에서 이어서 작업하는 방법

1. OneDrive 동기화가 완료되었는지 확인한다.
2. Codex에서 이 프로젝트 폴더를 연다.
3. 새 대화에서 `AGENTS.md와 CURRENT_WORK.md를 읽고 기존 작업을 이어서 진행해줘`라고 입력한다.
4. 작업 후 GitHub Desktop에서 변경 파일을 확인하고 Commit과 Push를 진행한다.
